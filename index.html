Selection Criteria & Vendors

Internal CPU Architecture: EU + BIU

Modern processors pipeline two units to increase efficiency:

Execution Unit (EU)

Contains the ALU, flags, and general registers. Decodes and executes instructions. Requests data from the BIU.

Bus Interface Unit (BIU)

Contains segment registers, instruction pointer, and a queue. Fetches instructions from memory ahead of time so the EU never has to wait.

Loops & DJNZ

The primary looping instruction in 8051 assembly and how to handle counts beyond 256.

DJNZ — Decrement & Jump if Not Zero

  • Decrements the specified register by 1
  • If result ≠ 0 → jump to label
  • If result = 0 → fall through to next instruction
Max iterations: 8-bit register max = 255 (FFH). If initialized to 0, first decrement wraps to 255 → 256 iterations max per single loop.

Nested Loop (700 iterations)

; Loop 700 times (10 × 70) MOV A, #55H MOV R3, #10 ; Outer = 10 NEXT: MOV R2, #70 ; Inner = 70 AGAIN: CPL A DJNZ R2, AGAIN DJNZ R3, NEXT
; Simple loop: Add 3 to A, 10 times MOV A, #0 MOV R2, #10 AGAIN: ADD A, #03 DJNZ R2, AGAIN

Stack & Subroutines

CALL, RET, PUSH, POP — and the LIFO stack that makes subroutines work.

CALL Instructions

LCALL3-byte. Target anywhere in 64 KB ROM.
ACALL2-byte. Target within 2 KB (memory efficient).
When CALL executes, the CPU pushes the 16-bit PC of the next instruction onto the stack (low byte first, then high byte), then jumps. RET pops it back.

PUSH / POP Rules

  • Must specify direct address, not register name. e.g., PUSH 4 for R4.
  • POP must be in reverse PUSH order (LIFO).
  • Stack Pointer (SP) resets to 07H on startup.

16-bit Arithmetic ADDED

When adding 8-bit numbers exceeds 255, the Carry flag allows you to build 16-bit results using two registers.

The Carry Flag (CY) in Arithmetic

When the result of an ADD operation exceeds FFH (255), the Carry flag is automatically set to 1 by the hardware. This is how 8-bit processors handle numbers larger than 8 bits.

Example:
0xF5 (245) + 0xE2 (226) = 0x1D7 (471)
A = 0xD7, CY = 1 (the overflow)

Use JNC to conditionally increment the high byte register (R5) only when overflow occurred.

16-bit Addition Code

; Add 0xF5 + 0xE2 into R5:R0 (16-bit result) MOV A, #0F5H ADD A, #0E2H ; A=D7H, CY=1 JNC NEXT ; No carry → skip INC R5 ; High byte += 1 NEXT: MOV R0, A ; Low byte in R0 ; R5:R0 = 0x01D7 = 471 decimal
ADDC (Add with Carry) is used in multi-byte chained additions to propagate the carry bit automatically.

Generating Square Waves ADDED

Using timers to generate accurate frequencies. Mode 2 (auto-reload) vs Mode 1 (manual reload).

✅ Mode 2 — Auto-Reload (Preferred)

Set a negative reload value in TH0. Every TL0 overflow automatically reloads and sets TF0. Just clear the flag and toggle the pin. Timer never stops.

; 10 kHz square wave on P1.0 ; Mode 2: TMOD = 0x02, TH0 = -50 = 0xCE MAIN: MOV TMOD, #02H MOV TH0, #-50 ; Reload = 0xCE MOV TL0, #-50 SETB TR0 LOOP: JNB TF0, LOOP CLR TF0 CPL P1.0 ; Toggle SJMP LOOP

⚠ Mode 1 — 16-bit (Manual Reload)

Needed for longer delays (e.g., 500 µs for 1 kHz). Because Mode 1 does NOT auto-reload, you must manually reset TH0:TL0 in software after every overflow — which can introduce slight timing inaccuracies.

; 1 kHz: 500µs interval LOOP: MOV TH0, #0FEH ; Must reload manually MOV TL0, #083H SETB TR0 HERE: JNB TF0, HERE CLR TF0 CLR TR0 CPL P1.0 SJMP LOOP
The reload time (2 MOV instructions) adds ~2 µs latency to each cycle, degrading frequency accuracy.

Intruder Alarm Scenario

Multiple interrupts working together: a door switch triggers a timed 1-second alarm.

🚨 Execution Sequence

State 0: Idle
Main configures INT0 for falling-edge. EA and EX0 enabled. CPU at SJMP $.
State 1: The Trigger
Door opens. INT0 drops to 0. CPU instantly jumps to vector 0x0003 (EX0ISR).
State 2: EX0ISR Runs
ISR enables ET0 & ET1, sets TR0 & TR1 to 1, returns with RETI.
State 3: Timers Active
Timer 1 ISR: Overflows rapidly, toggles P1.7 at 400 Hz (buzzer).
Timer 0 ISR: Overflows every 50ms, counts to 20 = 1 second.
State 4: Shutdown
Timer 0 reaches 20 overflows. Disables both timers, silences buzzer, resets.

Assembly Framework

ORG 0000H LJMP MAIN ; Skip vectors on reset ORG 000BH ; Timer 0 vector LJMP T0ISR ORG 0003H ; Ext INT0 vector LJMP EX0ISR ORG 0030H ; Main program above vectors MAIN: SETB IT0 ; Falling-edge trigger MOV IE, #81H; EA=1, EX0=1 SJMP $ ; Idle — wait EX0ISR: MOV IE, #9BH ; Enable all SETB TR0 SETB TR1 RETI T0ISR: DJNZ R7, DONE CLR TR0 ; Shutdown after 1s CLR TR1 DONE: RETI
IT0/IT1 External Interrupt Triggering ADDED

The TCON register bits IT0 and IT1 control how external interrupts are triggered. This is a critical concept often missed.

IT0/IT1 = 0 — Level Triggered

The interrupt is active as long as the pin stays LOW. If the pin is still low when the ISR finishes (RETI), the interrupt will fire again immediately.

⚠ Must ensure the source is de-asserted before RETI, otherwise the ISR triggers in an infinite loop.

IT0/IT1 = 1 — Falling-Edge Triggered

The interrupt fires exactly once when the pin transitions from HIGH → LOW. The CPU samples the pin every 12 clock cycles.

✅ Preferred for most applications — single, clean trigger per event.

IE0/IE1 — The Interrupt Edge Flag

When the 8051 detects a valid external interrupt event, hardware automatically sets IE0 (or IE1) to 1. When the CPU vectors to the ISR, this flag is automatically cleared (for edge-triggered). For level-triggered mode, it reflects the current pin level.

; Configure INT0 for falling-edge in TCON SETB IT0 ; IT0 = 1 → falling-edge mode SETB EX0 ; Enable External 0 interrupt SETB EA ; Enable global interrupts ; For level-triggered mode: CLR IT0 ; IT0 = 0 → level-triggered
